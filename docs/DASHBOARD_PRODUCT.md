# 仪表盘产品定位与改造说明

> 从产品经理视角对「仪表盘」的现状、问题与改造方向做简要说明，便于与 2.0 规划对齐。

---

## 一、现状与问题

### 当前仪表盘内容

- 标题「仪表盘」+ 主按钮「一键同步数据」
- 四个统计卡片：总数据集、就绪数据集、处理中、训练任务数
- 「最近数据集」表格（前 5 条）

### 存在的问题

1. **与业务页重复**：数据与训练任务数在各专项页（数据集管理、模型训练）已有完整列表，仪表盘仅做「数字汇总」，没有提供独特信息或决策依据。
2. **缺少「下一步」引导**：用户进入系统后不知道「该先做什么、后做什么」，仪表盘没有承担「工作台入口」的职责。
3. **「最近数据集」价值有限**：等同于数据集列表前 5 行，无快捷操作（如「用该数据集创建训练」），且与 2.0 的「流水线 / 可追溯」叙事脱节。
4. **一键同步权重过高**：作为低频运维能力，放在首页主 CTA 容易喧宾夺主，更适合放在设置或工具区。

---

## 二、项目背景下的定位建议

### 核心链路

- **数据 → 清洗 → 训练 → 评估**，经典版为手动逐步操作，Agent 版为一键流水线。
- 仪表盘是**经典版首页**，应成为「指挥台」：告诉用户当前处于链路的哪一阶段，并推荐下一步操作。

### 建议定位

- **名称**：可保留「仪表盘」或改为「工作台」。
- **职责**：
  - **推荐下一步**：根据当前数据状态（有无就绪数据集、有无进行中训练、有无已完成模型等）给出一条明确的「建议操作」及跳转（如「去上传数据」「创建训练任务」「查看训练进度」「去创建评估」「试试 Agent 版」）。
  - **资源概览**：保留少量核心数字（数据集 / 训练任务等），但改为可点击入口，点击进入对应模块，弱化「纯数字」感。
  - **最近动态**：用「最近动态」替代「最近数据集」——合并数据集、训练任务、流水线实例的最近记录，按时间排序，每条带类型标签与跳转，体现「数据 + 训练 + 流水线」一体。
  - **工具收口**：「一键同步数据」移至页脚或次要区域，不作为主 CTA。

---

## 三、改造后的信息架构（简要）

| 区域         | 内容说明 |
|--------------|----------|
| 推荐下一步   | 一条主建议卡片：文案 + 主按钮（如「去上传数据」「创建训练任务」「查看进度」「试试 Agent 版」），根据状态自动计算。 |
| 资源概览     | 4 个统计卡片（可点击进入对应页）：总/就绪数据集、训练任务、流水线数（可选），视觉弱于推荐卡片。 |
| 最近动态     | 合并展示最近的数据集/训练/流水线记录（如 5–8 条），每条：类型图标 + 名称/简述 + 状态 + 时间，点击跳转详情或列表。 |
| 工具         | 页底或卡片内小字链接：「从磁盘恢复数据」等。 |

---

## 四、与 2.0 的衔接

- 仪表盘不替代「流水线历史」或 Agent 画布，而是**经典版入口**：让用户快速判断「我现在该做什么」，并可通过「试试 Agent 版」引导至一键流水线。
- 若后续有「工作流模板」「Skill」等，可在「最近动态」旁增加「推荐工作流」或入口到 Agent 版/流水线历史。
