## 项目定位
- [系统] 训练完成
- [AI] 模型效果一般，建议增加数据量

---

## Agent 设计原则

### Data Agent
Data Agent = 确定性脚本分析 + 模板匹配 + LLM总结

职责：
- 基于用户上传数据做可靠性与可训练性分析
- 输出任务类型、问题、建议、基础统计

不负责：
- 自动采集外部数据
- 自动生成全量训练集
- 替用户包办数据构建

### Evaluation Agent
职责：
- 读取训练结果
- 生成自然语言解释
- 给出优化建议

不负责：
- 自动调参
- 自动多轮实验搜索

---

## 推荐开发顺序
1. 保持经典版可运行
2. 去掉 Planning Agent
3. 重构 Agent版 UI 为 4 阶段流程
4. 接入最小可用 Data Agent
5. 接入 Evaluation Agent
6. 最后做日志和体验增强

---

## Codex 工作方式要求
每次改动必须说明：
1. 本轮修改了什么
2. 为什么这么改
3. 对经典版兼容性做了什么保护
4. 剩余风险点是什么
5. 下一步建议是什么

如果存在不确定点，优先选择：
- 少改
- 兼容
- 可回退
- 可演示

不要为了“看起来像 multi-agent”而增加无必要复杂度。
