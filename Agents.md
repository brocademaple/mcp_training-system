## 双版本定位（产品与架构叙述）

- **经典版**：数据集、训练、评估的 **REST/页面操作面**，沉淀可查询的任务与数据状态；在毕设/产品叙述上可作为 **Agent 侧经 MCP 读取或衔接的数据源**（独立 MCP Server 的接口与部署由后续设计）。
- **Agent 版**：以 **MCP 协调的多 Agent** 为主线，侧重 **训练任务策划与流水线自动执行**（清洗→训练→评估），与经典版共用同一存储与 API 语义。

完整梳理（含独立 MCP Server 规划示例与代码索引）见：**[docs/AGENT_AND_MCP_GUIDE.md](docs/AGENT_AND_MCP_GUIDE.md)**。

## Agent 设计原则

### Data Agent
Data Agent = 确定性脚本分析 + 模板匹配 + LLM总结

职责：
- 基于用户上传数据做可靠性与可训练性分析
- 输出任务类型、问题、建议、基础统计

不负责：
- 自动采集外部数据
- 自动生成全量训练集
- 替用户包办数据构建

### Evaluation Agent
职责：
- 读取训练结果
- 生成自然语言解释
- 给出优化建议

不负责：
- 自动调参
- 自动多轮实验搜索

## 推荐开发顺序
1. 保持经典版可运行
2. 去掉 Planning Agent
3. 重构 Agent版 UI 为 4 阶段流程
4. 接入最小可用 Data Agent
5. 接入 Evaluation Agent
6. 最后做日志和体验增强

---

## Codex 工作方式要求
每次改动必须说明：
1. 本轮修改了什么
2. 为什么这么改
3. 对经典版兼容性做了什么保护
4. 剩余风险点是什么
5. 下一步建议是什么

如果存在不确定点，优先选择：
- 少改
- 兼容
- 可回退
- 可演示

不要为了“看起来像 multi-agent”而增加无必要复杂度。
